# 🌐 Web Development Topics

## 🏨 Hostel Room Allocation System

Build a fullstack system to manage hostel rooms and student allocations.

Requirements:

- Admin & Student dashboard
- Authentication system (Admin + Student roles)
- CRUD operations for rooms and students
- Room allocation with capacity and gender restrictions
- Track room occupancy and vacancy in real time
- Maintain allocation history and room change/vacate requests

---

## 📦 Inventory Management System

Build a fullstack system to manage products, stock, and transactions.

Requirements:

- Admin dashboard
- Authentication system (Admin role)
- CRUD operations for products and suppliers
- Track incoming and outgoing stock
- Maintain transaction history
- Implement low stock alerts and stock summaries

---

## ❤️ Tinder-Style PG Roommate Matching System

Build a fullstack system where users can find compatible roommates.

Requirements:

- User dashboard 
- Authentication and user profiles
- Store preferences (budget, lifestyle, habits)
- Filtering and profile browsing
- Mutual match logic
- Maintain matches and swipe history

---

# 🤖 Machine Learning Topics

## 🧾 AI-Powered Fake Bill Detection System

Build a system that analyzes uploaded invoices and detects potential fraud.

Requirements:

- Authentication and bill uploads (JPG/PDF)
- Extract invoice data using OCR
- Validate GSTIN, subtotal, tax and totals
- Detect duplicate invoices
- Generate fraud risk score

---

## 📰 AI-Powered Fake News Detection System

Build a system that classifies news content as Real or Fake.

Requirements:

- Authentication and article submission
- NLP preprocessing and text cleaning
- Feature extraction (TF-IDF or similar)
- Train ML classification model
- Store predictions and confidence scores

# 📖 README Requirements

Your repository must include a proper README explaining:

- What your project does
- How to install dependencies
- How to run the project
- How to use the system

Write instructions clearly so even a non-technical person can follow.

# 📊 Scoring System

### Commits — 10 Points

Commits should represent meaningful milestones.

Examples:

setup project structure  
implemented authentication  
completed dashboard UI  
fixed login validation bug

More commits does NOT mean more points.

---

### Presentation — 5 Points

After 8 days you will present your project.

Duration: 10 minutes  
Platform: Discord VC or Google Meet

⚠️ Mandatory. If you don't attend, you will be disqualified.

---

### Project Execution — 15 Points

Evaluation will consider:

- Feature completion
- Functionality
- Code organization
- Overall effort
